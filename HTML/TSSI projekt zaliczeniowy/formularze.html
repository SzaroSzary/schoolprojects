Formularze

Formularze są swoistym pomostem między trzema rodzajami języków - językiem znaczników, językiem przetwarzania po stronie serwera (server-side) i językiem przetwarzania po stronie użytkownika (client-side).

Znaczniki dotyczące formularzy:

  • <form> - znacznik kontenerowy, tworzy formularz, do jego właściwości należy:
    • action - definiuje rodzaj akcji, jaka nastąpi po wciśnięciu przycisku "Ok"
    • method - metoda przesyłania formularza, występuje w dwóch wartościach: "post" i "get" (domyślnie)
    • autocomplete - przeglądarka zapamiętuje lub nie dane wprowadzane wcześniej do formularza (domyślnie "on")
    • novalidate - wyłącza domyślne sprawdzanie zawartości
  • <textarea> - tworzy wieloliniowe pole tekstowe, które nie ma określonej maksymalnej ilości znaków
    • rows - definiuje jaką wysokość ma mieć <textarea> (jeden wiersz = 1)
    • cols - definiuje jaką szerokość ma mieć <textarea> (jeden znak = jedna kolumna)
  • <label> - definiuje etykietę do poszczególnych pól <input/>
  • <input/> - jest to znacznik, który umożliwia użytkownikowi wprowadzanie danych do formularza
    • accept - określa typ plików, które akceptuje serwer
    • autocomplete - określa, czy pole powinno być automatycznie wypełniane
    • autofocus - określa które pole powinno być domyślnie aktywne przy odświeżeniu strony
    • checked - określa czy pole powinno być domyślnie zaznaczone (jedynie dla typów "checkbox" oraz "radio"
    • disabled - określa, czy pole powinno być wyłączone
    • list - korzysta z zawartości elementu <datalist>
    • max - określa maksymalną wartość dla pola (dotyczy numerów i dat)
    • maxlength - określa maksymalną ilość znaków możliwych do zastosowania
    • min - określa minimalną wartość dla pola (dotyczy numerów i dat)
    • placeholder - podaje krótką wskazówkę na temat spodziewanej wartości wprowadzonej do pola
    • readonly - określa czy pole jest tylko do odczytu
    • required - określa czy pole jest wymagane, można dzięki niej stworzyć pseudoklasę :required (tak samo przy :optional)
    • size - określa ilość wyświetlanych znaków w polu wprowadzania
    • src - określa obrazek użyty jako przycisk zatwierdzenia
    • tabindex - określa kolejność przełączania pól za pomocą klawisza tab
    • type
      • button - określa przycisk
      • checkbox - określa pole wyboru, które dopuszcza wybranie wielu wartości
      • color - określa pole wyboru koloru
      • date - określa pole wyboru, które powinno zawierać datę
      • email - określa pole tekstowe, które powinno zawierać adres e-mail
      • file - określa plik do przesłania, wymaga zamieszcenia obok znacznika <form> linijki method="post" enctype="multipart/form-data"
      • hidden - ukrywa pole formularza
      • image - określa obrazek, który będzie działał jak przycisk submit
      • number - używany przy polach wymagających wartości numerycznych
      • password - określa pole wprowadzania hasła
      • radio - określa pole wyboru, które dopuszcza wybranie tylko jednej wartości
      • range - określa suwak reprezentujący jakiś zakres
      • reset - resetuje wpisane do formularza dane
      • submit - określa przycisk zatwierdzania formularza
      • tel - używane do wprowadzania numeru telefonu
      • text - określa pole wprowadzania tekstu
      • url - używane do wprowadzania adresu URL
  • <button> - tworzy przyciski, mogą one być trzech typów:
    • submit - zatwierdza dane formularza, jest równoważny polu input typu submit
    • reset - resetuje dane formularza, jest równoważny polu input typu reset
    • button - sam w sobie nie robi nic, ale może zostać zaprogramowany, jest równoważny polu input typu button
  • <select> - znacznik kontenerowy, tworzy menu kontekstowe z opcjami do wyboru
  • <option> - określa poszczególne opcje menu kontekstowego
  • <optgroup> - dzieli poszczególne opcje menu kontekstowego na grupy
  • <fieldset> - tworzy ramkę dookoła formularza
  • <fieldset> - tworzy podpis formularza umieszczony w ramce
    Opis
  • <datalist> - tworzy listę opcji, które są możliwe do wyboru
